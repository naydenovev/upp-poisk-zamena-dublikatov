## ADDED Requirements

### Requirement: Область кандидатов на удаление
Обработка SHALL ограничивать кандидатов на пометку и удаление исключительно элементами текущей группы результатов поиска и замены (реквизит формы `НайденныеОбъекты`, строки с `Правильный = Ложь`), полученными в текущем сеансе поиска/замены. Обработка SHALL NOT обращаться к глобальному списку помеченных на удаление объектов базы (`Обработки.УдалениеПомеченныхОбъектов.ПолучитьПомеченныеНаУдаление`) для определения кандидатов.

#### Scenario: Кандидаты берутся только из текущей группы
- **WHEN** пользователь выполнил поиск и замену дублей для группы N и нажимает «Удалить элементы»
- **THEN** обработка помечает на удаление и передаёт на контроль только ссылки из группы N (`Правильный = Ложь`), не затрагивая объекты, не входившие в результаты поиска

#### Scenario: Ранее помеченные объекты вне поиска не затрагиваются
- **WHEN** в базе есть объекты, помеченные на удаление другим пользователем или процессом и не входящие в текущие результаты поиска
- **THEN** операция «Удалить элементы» не включает эти объекты в контроль и удаление

### Requirement: Пометка на удаление перед контролем
Обработка SHALL помечать каждый элемент-кандидат на удаление (`ПометкаУдаления = Истина`, запись объекта) до выполнения проверки возможности удаления.

#### Scenario: Элементы помечаются перед контролем
- **WHEN** пользователь подтвердил удаление элементов текущей группы
- **THEN** для каждого элемента-кандидата обработка устанавливает `ПометкаУдаления = Истина` и записывает объект до вызова контроля удаления

### Requirement: Контроль удаления через типовой механизм
Обработка SHALL выполнять контроль возможности удаления и собственно удаление через типовой `Обработки.УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты`, реализующий тот же алгоритм проверки ссылочной целостности, что и штатная обработка «Удаление помеченных объектов». Обработка SHALL NOT реализовывать собственную независимую логику проверки ссылочной целостности взамен типовой.

#### Scenario: Успешное удаление элемента без внешних ссылок
- **WHEN** элемент-кандидат помечен на удаление и на него не осталось ссылок в базе
- **THEN** типовой механизм удаляет элемент, и он пропадает из списка результатов поиска обработки

#### Scenario: Элемент не может быть удалён из-за оставшихся ссылок
- **WHEN** элемент-кандидат помечен на удаление, но на него ссылается другой объект базы
- **THEN** типовой механизм не удаляет элемент; элемент остаётся помеченным на удаление (`ПометкаУдаления = Истина`) и не пропадает из списка результатов поиска обработки

### Requirement: Подтверждение перед необратимой операцией
Обработка SHALL запрашивать явное подтверждение пользователя перед пометкой и удалением, с указанием количества элементов-кандидатов.

#### Scenario: Пользователь подтверждает удаление
- **WHEN** пользователь нажимает «Удалить элементы» и в диалоге подтверждения выбирает «Да»
- **THEN** обработка выполняет пометку и контролируемое удаление кандидатов текущей группы

#### Scenario: Пользователь отменяет удаление
- **WHEN** пользователь нажимает «Удалить элементы» и в диалоге подтверждения выбирает «Нет»/«Отмена»
- **THEN** обработка не помечает и не удаляет ни одного элемента, состояние результатов поиска не меняется

### Requirement: Отчёт о результате
Обработка SHALL после завершения операции показывать пользователю отчёт с количеством помеченных, успешно удалённых и не удалённых элементов; для не удалённых — информацию о том, что послужило препятствием (найденные ссылающиеся объекты), если эта информация доступна из результата типового механизма.

#### Scenario: Отчёт при частичном удалении
- **WHEN** из группы кандидатов часть элементов удалена, а часть — нет из-за оставшихся ссылок
- **THEN** пользователю показывается итог с числом удалённых и числом неудалённых элементов и данными о блокирующих ссылках для неудалённых

### Requirement: Отсутствие изменений в пакетном пути «Автозамена»
Обработка SHALL сохранять текущее поведение процедуры массовой автозамены (`АвтозаменаЭлементов`, кнопка «Автозамена») без изменений в рамках этого требования.

#### Scenario: Автозамена не использует новый механизм
- **WHEN** пользователь выполняет пакетную автозамену с опцией немедленного удаления
- **THEN** удаление в этом сценарии выполняется существующим способом (прямое удаление объекта), без пометки на удаление и без вызова типового `УдалитьПомеченныеОбъекты`
