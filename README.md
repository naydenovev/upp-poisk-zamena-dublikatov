# Поиск и замена дублирующихся элементов + пометка и удаление дублей

Доработка типовой обработки 1С:УПП (Украина, ред. 1.3) «Поиск и замена дублирующихся
элементов справочников», сохранённой как внешняя обработка.

## Что добавлено

После выполнения поиска и замены дублирующихся элементов справочника кнопка
**«Удалить элементы»** (вкладка «Найденные ссылки») теперь:

1. Запрашивает подтверждение у пользователя с указанием количества элементов.
2. Помечает элементы-дубли текущей группы на удаление (`ПометкаУдаления = Истина`).
3. Выполняет контроль возможности удаления и собственно удаление через типовой
   механизм `Обработки.УдалениеПомеченныхОбъектов.УдалитьПомеченныеОбъекты(...)` —
   тот же алгоритм, что использует штатная обработка «Удаление помеченных
   объектов» (итеративная проверка ссылочной целостности).
4. Показывает отчёт: сколько элементов помечено, удалено, не удалено (и что
   именно на них ссылается, если удалить не удалось).

Кандидатами на пометку и удаление являются **только** элементы, найденные и
обработанные в текущем сеансе поиска/замены этой обработки — обработка не
сканирует и не трогает объекты, помеченные на удаление кем-то другим.

Элементы, которые типовой механизм не смог удалить (остались ссылки),
остаются помеченными на удаление — для последующей обработки штатной
«Удаление помеченных объектов».

Пакетный режим «Автозамена» не изменён и сохраняет прежнее поведение.

## Структура репозитория

- `ПоискИЗаменаДублирующихсяЭлементов.epf` — исходная (немодифицированная)
  внешняя обработка, сохранённая из типовой конфигурации УПП Украина 1.3.
- `ПоискИЗаменаДублирующихсяЭлементов.new.epf` — доработанная обработка.
- `src_epf/` — исходники обработки в текстовом виде (распакованы через
  [`v8unpack`](https://github.com/saby/v8unpack)), правки внесены здесь
  (`Form/Форма/Form.obj.bsl`).
- `obr_ПоискИЗаменаДублирующихсяЭлементов_xml/` — выгрузка обработки конфигуратором
  в XML (справочный материал, обычные формы там остаются бинарными).
- `openspec/` — спецификация доработки (proposal/design/specs/tasks),
  оформленная через [OpenSpec](https://github.com/Fission-AI/OpenSpec).

## Требования к целевой базе

Доработка предполагает наличие в целевой конфигурации типового объекта
`Обработки.УдалениеПомеченныхОбъектов` (стандартно присутствует практически
во всех конфигурациях 1С, включая УПП Украина 1.3). При его отсутствии
обработка не удаляет объекты и выводит соответствующее сообщение.
